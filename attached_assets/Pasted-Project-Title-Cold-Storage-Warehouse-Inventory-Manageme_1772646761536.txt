Project Title:
Cold Storage Warehouse Inventory Management System (Internal MVP)
Project Type:
Web-based internal inventory management system for agricultural cold storage (Potato & Corn). 
Tech Requirements: Web application
Role-based authentication, Optimized for 25 concurrent users, High-performance for large datasets and Scalable design
Business Context: This system manages cold storage warehouses storing potatoes and corn.
Operations scale:
~250 trucks received per day
~20 warehouse locations
~40 cold rooms per warehouse
Lot traceability required
Partial outbound shipments allowed
This is a lot-based inventory system (not product-only inventory).
CORE FEATURES
1️⃣ Authentication & Roles
Create role-based access control:
Roles:
Admin (full access)
Manager (warehouse-level management)
Data Entry (can create/update records but not delete)
Viewer (read-only access)
Each user may optionally be restricted to a specific warehouse.
Secure internal login system.
2️⃣ Warehouse Structure
Admin can dynamically create:
1-Warehouse:
Name
Location
2-Cold Rooms:
Linked to warehouse
Type of potato to be stored
Fixed capacity (kilo)
Storage type (bulk / pallets )
Active/Inactive status
The system must calculate room utilization automatically.
3️⃣ Supplier & Farm Management
Track:
Supplier
Farm source
Contact information
4️⃣ Inbound Operations (Truck-Based)
When a truck arrives, system records:
Truck number
Supplier
Farm source
Product type (Potato / Corn)
Variety type
Anti-sprout type
Total weight (tons)
Arrival date
The system generates a unique Lot Number (Batch ID).
Then assign batch to:
Warehouse
Room
Track:
Kilograms
Jampo bag numbers
Expiration date
Received date
Status (Stored / Partial / Depleted / Expired)
5️⃣ Outbound Shipments
Allow:
Full batch shipment
Partial shipment
Track:
Batch (lot number)
Quantity shipped (tons)
Destination
Shipped date
Remaining quantity auto-calculated
6️⃣ Loss Tracking
Allow recording:
Rot
Damage
Shrinkage
Fields:
Batch ID
Loss type
Quantity
Recorded date
Remaining stock auto-adjusted.
7️⃣ Sensor Monitoring
Each cold room logs:
Temperature
Humidity
CO2 level
Timestamp
Sensor data stored efficiently (designed for large growth).
Dashboard must highlight:
Rooms exceeding safe thresholds.

Generate:
Daily inbound report
Daily outbound report
Inventory snapshot report

CSV
